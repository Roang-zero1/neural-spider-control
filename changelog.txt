Version: 2.5.1
Date: 2026-05-13
  Changes:
    - Reconnect to last vehicle shortcut is now Shift+Z (was Ctrl+Z), so it stops fighting undo in chat and elsewhere.
    - Hooks into Vehicle Control Center using its current remote name (vehicle-control-center) and the new context-button API instead of the old “centre” spelling and simple register-button flow.
    - ceelos-vehicle-gui-util is no longer required: the left-toolbar and vehicle GUI actions (neural connect, orphaned engineer inventory, call spidertron, map at vehicle, open vehicle inventory) are registered and refreshed through Vehicle Control Center.
    - Neural connect only shows when it makes sense: empty seat, or only a reusable dummy/orphan in the driver slot; hidden while the vehicle already has an active neural link or a real player in the seat.
    - Works more cleanly when you’re on the spidertron remote: selection is resolved the same way Vehicle Control Center expects, with clearer feedback if nothing is selected.
    - Dummy-engineer detection treats orphaned engineers the same as active ones where it matters, so inventories and UI logic stay in sync.
    - Fallback vehicle GUI helpers were adjusted for the newer toolbar layout and icons, and opening the map prefers Vehicle Control Center’s follow-on-map when that mod offers it.
    - Older neural/orphan button names from earlier layouts still route through the same actions, so you are less likely to end up with dead clicks after an upgrade.
    - Introduced new functions for vehicle entity comparison and player control validation.
    - Refactored vehicle resolution logic to streamline vehicle retrieval from payloads.
    - Added conditions for neural disconnect functionality, ensuring proper player-vehicle control checks.
    - Updated GUI integration to include a new "Neural Disconnect" button with appropriate context actions.
    - Enhanced refresh functionality for the Vehicle Control Center toolbar upon connection and disconnection events.
  Bugfixes:
    - On vehicle death, move dummy engineer inventory into the vehicle trunk (corpse) first, then spill overflow at the wreck — reverted after in-game test still lost dummy items.

Version: 2.5.0
Date: 2025-12-11
  Features:
    - Orphaned dummy engineer system: dummy engineers are kept alive after disconnection if they have items, are crafting, or have active logistic requests
    - Delayed destruction: dummy engineers are not immediately destroyed on disconnect, allowing crafting to continue and preventing immediate item spillage
    - Smart reconnection: reconnecting to the same vehicle reuses the existing orphaned dummy engineer instead of creating a new one
    - GUI integration: added button in vehicle GUI to open the inventory of connected/orphaned dummy engineers
    - Physical entry handling: when a player physically enters a vehicle with an orphaned dummy engineer, the engineer is automatically destroyed and items are spilled
    - Periodic cleanup: automatic cleanup of completely idle orphaned engineers (no items, no crafting, no logistic requests) every 15 minutes
    - Vehicle destruction handling: orphaned engineers are properly cleaned up when their associated vehicle is destroyed
  Bug fixes:
    - Improved connection stability and reliability
    - Enhanced entity reference restoration after game load
    - Optimized orphaned engineer cleanup logic
    - Refined GUI button integration with Vehicle Control Center
    - Better handling of edge cases in connection/disconnection scenarios
